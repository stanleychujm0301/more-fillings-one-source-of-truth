## 二、普通 AI 方案 vs 我们（1 分钟）

**主讲：** "有人会说，扔给 ChatGPT 不就行了？我们对比一下："

| 对比维度 | 直接喂 ChatGPT | 我们的方案 |
|---|---|---|
| 数据精度 | 数字会幻觉、单位错 | 规则引擎做硬校验，0 幻觉 |
| 准则差异 | 不知道"这本就该不一样" | RAG 准则库智能解读，附引用条款 |
| 证据链 | 无页码、无定位 | 每条差异附 PDF 高亮跳转 |
| 数据合规 | 出境风险，KPMG 内规不允许 | 国产 LLM + 可本地部署 |
| 图表识别 | 不读图 | Qwen-VL 抽数据，图表/表格/文字三方核对 |

## 三、Demo 三场景（3-4 分钟）

### 场景 1：数值差异检测（60 秒）

**操作**：上传演示用的 A 股 PDF + H 股 PDF，按"开始检查"。

**呈现**：

1. 进度条从 0% 走到 100%，标注：**实际仅 28 秒，人工需 8 小时**
2. Excel 报告自动弹出，第一条差异：
   > 总资产差异 5,234 万元（A 股 P.45 vs H 股 P.38）严重度: CRITICAL
3. 点击该行 → 右侧 PDF 预览自动跳转到 P.45，红框高亮"资产总计 12,345,678 万元"

**主讲台词**："每条差异都能 1 秒跳到原报告，审计师不需要翻页对照。"

### 场景 2：准则差异智能解读（90 秒，亮点 1）

**操作**：点击差异列表中的"研发支出"行。

**呈现**：

- 差异：A 股研发资本化 1.20 亿，H 股 0.95 亿，差异 0.25 亿（21%）
- 普通方案：标"重大差异，需追问"
- 我们的方案展示一段 AI 推理：

> **AI 解读（智能检索准则）**：
> 该差异 21% 处于 CAS 6 与 IAS 38 资本化时点判断差异的合理范围内，**符合预期**。
>
> 引用准则：
> 1. CAS 6 第 9 条：开发阶段资本化需"同时满足 5 个条件"
> 2. IAS 38 Paragraph 57：开发阶段判定即可资本化
> 3. KPMG《IFRS compared to CAS》Section 2.3
>
> 置信度：92%

**主讲台词**："这是从『AI 比对工具』升级为『AI 审计助理』的关键 — 不只发现问题，还能给出专业判断。"

### 场景 3：图表交叉核对（60 秒，亮点 2）

**操作**：切换到"图表核对"标签。

**呈现**：

- 年报 P.20 一张饼图："2024 业务收入构成"
- 系统从饼图识别：零售业务 = 36.2%、对公业务 = 45.5%、其他 = 18.3%
- 同页表格：零售业务 = 35.8%
- 同页文字陈述："零售业务占 35%"
- 系统标记 **"三方不一致警告"**：偏差 1.2%，建议复核数据来源

**主讲台词**："这种细微不一致，人眼几乎查不出来 — 现在 AI 一秒发现。"

## 五、Q&A 预设（评委可能问）

### Q1: LLM 幻觉怎么办？
**A**: 数值检查走规则引擎不走 LLM；LLM 仅做语义对齐和准则解读，每条都附 RAG 引用源，并标"AI 建议"。confidence < 0.85 直接降级为待人工复核。

### Q2: 国产 LLM API 的数据安全？
**A**: 通义千问/智谱企业版均承诺数据不入训练；架构上可一键切到私有化部署的 Qwen2.5 14B（已演示）。

### Q3: 扫描件年报怎么处理？
**A**: PaddleOCR ch_PP-OCRv4 兜底，中文识别准确率 95%+。我们已对老旧年报样本做过测试。

### Q4: 准则更新跟得上吗？
**A**: 知识库是 Markdown 文件，审计师 5 分钟新增一条；提交后 30 秒重建 RAG 索引。**审计师不需要程序员介入**。

### Q5: 跟 KPMG Clara 的差异？
**A**: Clara 偏单文档审计流程内的工作底稿生成，我们专攻"**跨报告一致性**"这个 Clara 没覆盖的场景。

### Q6: 演示里只有 3 个场景，实际能力如何？
**A**: 后台已识别 47 条差异，因时间限制只演 3 类典型，详见 Excel 报告（评委 U 盘里有）。

### Q7: 你们怎么验证漏检率 3.2%？
**A**: 主办方提供的样本含预置答案，我们脚本 `scripts/eval_samples.py` 自动比对，召回率 96.8%。

### Q8: 准则差异库 15 条够吗？
**A**: 覆盖 80% 的 A+H 高频差异场景。剩余长尾差异用 LLM 通用推理兜底，并标"低置信度"。
